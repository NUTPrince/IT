# Git 的基础操作
## 修改、提交

首先先来理解下 Git 工作区、暂存区和版本库概念：

![[g-5.png]]

-   **工作区**：本地工作目录的文件夹

-   **版本库**：工作区有一个隐藏目录 .git，其是 Git 的版本库（`.git 文件夹`：初次对本地文件夹使用 `git init` 命令，即可得到）

-   **暂存区**：一般存放在 .git 目录下的 index 文件（.git/index）中

**把文件往Git版本库里添加的时候，是分两步执行的：**

1. 先使用`git status`查看查看整个仓库的状态，若有更改未提交，再使用`git add`把文件修改添加到暂存区

2. 用`git commit`提交更改到版本库（当前分支->main）
(在初始化创建Git版本库时，Git自动为我们创建了唯一一个`main`分支，所以，现在，`git commit`就是往`main`分支上提交更改。)

>总的来说就是，把需要提交的,修改后的文件全部放入暂存区（`git add .`或者`git add [文件名]` 命令），然后一次性提交暂存区的所有修改(`git commit -m "xxx"`命令)。

## 撤销修改

若要撤销暂存区的修改怎么办，则可执行 `git reset -- [文件名]` 即可撤销提交到暂存区的修改。

![[g-6.png]]

## 查看提交历史
使用命令 `git log`，它用来查看版本库的提交历史记录。

>执行 `git log` 查看提交记录时，十六进制序列号就是提交版本号，这是很重要的信息，每个提交都有自己单独的版本号，就像公民身份证号一样。

```shell
$ git log

> commit d98c961257dcc92863a5a792749b1e94caab69c2 (**HEAD ->** **main**, **origin/main**)
```

## 推送到 GitHub 远程仓库
使用命令：
```shell
git push
```
然后等待即可。

## 版本回退

发现文件内容有误需要修改，但是已经推送到版本库，怎么办？

**方法一：**

修改此文件然后再次添加到暂存区、提交、推送。

**方法二：**

撤销最近一次提交，修改文件后重新提交推送。

执行 `git reset --soft HEAD^` 撤销最近的一次提交，将修改还原到暂存区。`--soft` 表示软退回，对应的还有 `--hard` 硬退回，后面会讲到，`HEAD^` 表示撤销一次提交，`HEAD^^` 表示撤销两次提交，撤销 n 次可以简写为 `HEAD~n`。软退回一个提交后执行 `git branch -avv` 命令查看分支信息：
